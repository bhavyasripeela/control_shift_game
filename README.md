# Control Shift Game

A rule-breaking platform built for a Vibecoding Hackathon.

## Overview

This project explores the idea that the player is not always in control. The game begins as a simple platformer where the player controls the character to avoid obstacles. Midway through gameplay, the core rule changes — the character moves automatically, and the player must manipulate the environment to ensure survival.

## Core Concept

The game demonstrates a **rule-breaking mechanic** where control shifts from the character to the world. Instead of moving the hero directly, the player interacts with the environment by moving platforms, disabling obstacles, and clearing paths.

## Gameplay Flow

**Phase 1 – Traditional Control**

* Player moves the character.
* Jump across gaps and avoid spikes.
* Moving background creates a runner-style experience.

**Phase 2 – Rule Break**

* Character begins moving automatically.
* Player can no longer control movement.
* Instead, the player interacts with the environment to help the character survive.

## Interactive Mechanics

* Draggable platforms
* Tap-to-destroy obstacles
* Temporary obstacle disabling
  
* Particle effects for jumps and explosions

## Visual Design

The game uses contrasting visual styles to highlight the rule change:

* **Phase 1:** Calm blue environment with predictable obstacles.
* **Phase 2:** Dark neon environment with interactive elements and particle effects.
